What landed (and stays)

TOKEN DISCIPLINE

Every color a CSS var on :root + [data-theme="light"]. No hardcoded hex in component CSS. Workspace color flows via single --ws custom property. Exactly as briefed — this is the foundation.

TYPE STACK

Orbitron (display) + Rajdhani (ui) + Share Tech Mono (mono) + Press Start 2P (3P only). All locally-served, all OFL-licensed. Reads sci-fi without being neon.

CTX HEATMAP (NEW)

Not in my brief — they added traffic-light bands on the 10-bar context fill: ≤40% green, 40–75% yellow, 75%+ red. Keep this. Better than my flat-color spec.

ENGINEERING-GRID BG

32px minor / 160px major grid behind everything. Reads as graph-paper / blueprint. Subtle, restrained, exactly the sci-fi nod I asked for.

HUD BRACKETS

Corner brackets only on outermost frames (drawer outer, .nexus-app shell). Restraint per the no-glow rule. Got it right.

THEME TOGGLE + LIGHT MODE

Dark default + paper-on-tan light. localStorage persistence. Both modes look properly considered, not just inverted.

Unilateral calls — need your sign-off

1

Drawer collapsed = continuous ticker (Variant B). Bordered-tile variant dropped.

My v3 wireframes had both. They chose the ticker — 50px outer, 24px rows, no inter-row gaps, single horizontal color wash per row. Cleaner ambient density. I agree.

LOCK IT — confirm "ticker only"
2

Workflow-dot row dropped from expanded work view.

CSS exists (.wf .slot.standby/.human/.block), HTML doesn't render it. They flagged it as polish. This is wrong-shaped — the two-set badge model (engine + workflow) was load-bearing from your 2026-05-20 feedback. Workflow dots should ship in the default expanded view, empty slots showing as dashed placeholders.

ADD WORKFLOW DOTS BACK — default-on, even when empty
3

CL/CX marks = stylized house-style approximations, not the real Anthropic / OpenAI logos.

The PNGs in uploads/Claude.png + uploads/ChatGpt.png are the official marks they had access to. They chose stylized for trademark safety. Right call for shipped product — using official marks invites legal friction. Keep stylized; iterate the marks themselves if they don't read right.

KEEP STYLIZED — refine the glyphs, not the source
4

--ws-{name} tokens vs single --ws per workspace.

They enumerated 6 named tokens (overmind / hpfhq / debugs / scratch / bite / medus). Categories are user-defined and open-ended. Should be single --ws per workspace, applied via inline style or data attribute. Drop the named enum.
