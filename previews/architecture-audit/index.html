03Findings — what's actually worth fixing

HIGHNova identity confusion

Two things named "Nova": projects/nova/ is a placeholder with design docs only, while scripts/nova-sweep.js is the actual running system. The project dir's CONTRACT.md describes a future adapter that doesn't exist, which any new session reading it would treat as live.

Either move scripts/nova-sweep.js into projects/nova/ and make it the canonical home, or rename the project dir to projects/nova-plan/ so a fresh reader can't confuse intent with implementation.

HIGHDrone → Relay format-by-convention

drone-to-relay.js parses inbox/drone-<date>.md by regexing markdown headings produced by drone.js. If drone.js ever tweaks its output template, drone-to-relay.js silently drops entries — no error, just lost ideas.

Either (a) write a JSONL sidecar alongside the markdown report (drone-<date>.jsonl) and have drone-to-relay.js read that instead, or (b) extract the shared template into one module that both scripts import.

MEDIUMHPFHQ .env.local as Overmind's secret source

Drone, Sweep, Nova-sweep, and (indirectly) Signal Archive all fall back to reading ~/Code/hpf-command-center/.env.local when their own env isn't set. That's a structural dependency from Overmind into HPFHQ — opposite to the pollution direction your tier model expects (Overmind sees outward, not into a leaf tenant).

Already covered by the 2026-05-14 1Password migration ([[project_one_password_migration_2026_05_14]]). Audit which scripts still have the HPFHQ fallback path and remove — they should source from op run --env-file=<overmind-side>.op.env exclusively.

MEDIUMStreamdeck silent dep on external repo

Multiple Streamdeck scripts hardcode imports from /Users/emil/Code/streamdeck-cockpit/lib/sd_render.py. That repo isn't in overmind, isn't a submodule, and isn't documented as a dependency. A fresh clone of overmind on a new Mac won't run Streamdeck widgets.

Three options, in increasing effort: (a) add a one-line note in projects/streamdeck/README.md listing the external dep, (b) symlink the lib into projects/streamdeck/lib/ at install time, (c) git-submodule or vendored copy.

MEDIUMCockpit doesn't exist as a system

You talk about Cockpit as if it's a thing, but projects/cockpit/ is design docs. The actual operational dashboard is fragmented across projects/streamdeck/swiftbar/cockpit-bar.10s.sh, the Stream Deck buttons, and (eventually) ultrawinning.com pages. New work on "the cockpit" has no canonical home.

Decide: is Cockpit (1) the design-system folder, (2) the SwiftBar bar, or (3) the future ultrawinning.com surface? Name it explicitly and put a README in the canonical dir pointing to the others.

LOWSoft conflict on ~/Documents/Overmind/drone/

Drone reads + moves files out of this directory; capture-router writes new files into it (route=upgrades). The two never run simultaneously by accident (capture-router is event-driven, drone is launchd every 6h), but if they did, a partially-written file could be read mid-flight.

Lowest priority — the race is theoretically possible but hasn't happened. If it ever does: have capture-router write to a .tmp sibling and rename when done (atomic move pattern).

LOWauto-wrap-stop event format undocumented

The Claude Code hook at ~/.claude/hooks/auto-wrap-stop.js appends JSONL events to ~/.comsat/logs/sessionstart.log that COMSAT then tails. The event shape {event, workspaceId, ...} isn't documented anywhere except in the consuming code.

Add a short HOOK-CONTRACT.md to projects/comsat/ listing every external-write source that COMSAT consumes and the expected event shape per source.

04What's right (don't break these)

Three architectural choices that are doing real load-bearing work and should be preserved:

iCloud envelope queue as iPhone ↔ Mac bridge

Using iCloud Drive to hand markdown envelopes between iPhone and Mac is a clean physical contract — no API, no auth, no server. Both sides read/write the same folder. The fact that it works offline-first is a strong property.

WORM logs as the Player 3 spine

Append-only JSONL at ~/.claude/state/{codex,local}/log.jsonl is the canonical record of every 3p invocation. Consumers (COMSAT, usage-snapshot, status checks) all read the same file. This is the right pattern — copy it elsewhere when adding new agent lanes.

COMSAT as the only HTTP service in the stack

Streamdeck, Stream Deck Mobile, /wrap auto-allow, and future remote-control tools all go through one local HTTP API. One service to harden, one port to firewall, one auth model to think about. Resist the urge to add a second HTTP service for any new feature — extend COMSAT instead.
