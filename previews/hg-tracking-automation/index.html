HG Funnel Tracking — Automation Map

What plugs into APIs, what's calculated, what truly needs a human. Plus the onboarding steps for each integration.

v0.1 · 2026-05-06 Bite / Hypergrowth Verified against Meta Graph API v25 (May 2026)
TL;DR. ~95% of HG's funnel metrics can be pulled or derived from APIs. Three fields require unavoidable human input — and even those become single-tag clicks inside GHL once the pipeline schema is set up. The unlocks are: Meta Ads + IG Graph + GHL + Stripe (or GHL Payments) + a meeting platform integration. Optional: call recording + LLM scoring removes the last manual judgment fields.

Onboarding Steps — Per API

01Meta Ads API

Drives all paid-side metrics: spend, CPL, objective, audience type, creative-level performance.

  • Create a Meta Developer App at developers.facebook.com → Apps → Create.
  • Add the Marketing API product.
  • In Business Manager, generate a System User access token (long-lived, no 60-day expiry).
  • Grant scopes: ads_read, ads_management, business_management.
  • Save the token and the ad account ID (act_XXXXXX) into Bite env.
  • Endpoint pattern: GET /act_<id>/insights?fields=spend,actions,objective,reach&date_preset=this_month&level=ad.

02Instagram Graph API

Per-post profile visits and per-post follower attribution — both available at media level as of v25.

  • Convert client's IG account to Business or Creator (in the IG app → Settings → Account type).
  • Connect the IG account to a Facebook Page (required by API).
  • Reuse the Meta App from step 01; add the Instagram Graph API product.
  • Request scopes: instagram_basic, instagram_manage_insights, pages_show_list.
  • Submit for App Review for production traffic (skippable for own-account dev token).
  • Endpoint pattern: GET /<media_id>/insights?metric=profile_visits,follows,reach,views.
  • Verify before relying on it: test the follows field against one boosted post — confirm it returns post-level follower growth (matches the IG Professional Dashboard number) rather than total account followers. Field description in Meta docs is ambiguous.

03GoHighLevel API

Central system of record for setter activity, closer activity, deal stages, contracts.

  • In GHL → Settings → Private Integrations → create a Private Integration token.
  • Required scopes: contacts.readonly, opportunities.readonly, calendars/events.readonly, payments/transactions.readonly.
  • Required GHL setup before pulling is meaningful:
    • Pipeline with stages: Lead → Offer Made → Booked → Showed → Qualified → Closed Won.
    • Tag schema for setter/closer judgment fields (e.g. qualified, offer-made, show, no-show).
    • Setters and closers must update on mobile in real time, or via a Zap.
  • Endpoint: GET /opportunities/search?location_id=<id>&pipeline_id=<id>.

04Stripe (or GHL Payments)

Powers all cash-collected metrics and auto-flips deals to closed-won.

  • Stripe path: create a restricted API key with charges:read, customers:read, invoices:read.
  • Subscribe Bite webhook to charge.succeeded and invoice.paid.
  • GHL Payments path: already covered by the GHL token in step 03 (payments/transactions.readonly).
  • On payment event: write back to GHL deal → flip stage to Closed Won.

05Zoom or Google Meet — attendance

Eliminates manual show / no-show flagging.

  • Zoom: create an OAuth app with report:read:admin; pull /report/meetings/<meeting_id>/participants.
  • Google Meet: Workspace admin enables Meet attendance reports → pull via Google Reports API.
  • Match join logs to GHL appointments by attendee email or scheduled start time.
  • Below ~90s of attendance = no-show.

06Setter / DM tool — Manychat, Instantly, etc.

Captures OB volume and reply rate without manual logging.

  • Manychat: account API key → pull broadcast and sequence delivery + reply stats.
  • Instantly / similar: exports campaign-level send / reply / positive-reply counts.
  • If setters use the IG inbox manually with no tool: no API path. They must tag in GHL on offer-made.

07Call recording + LLM scoring (optional advanced layer)

Removes the last subjective field — the closer's "qualified Y/N" tag.

  • Record calls via Zoom Cloud Recording, Fathom, or Gong.
  • Pipe transcripts through an LLM with HG's qualification rubric.
  • Write the score back to the GHL deal as a tag.
  • Build cost: ~1 day eng, then ~$0.01–0.05 per call in LLM cost.

3. Truly 100% manual Irreducible

If you don't build the optional infrastructure flagged above, these stay manual. With the full stack in place — meeting-platform integration plus call recording plus LLM scoring — the irreducible list is empty.

MetricWhy manual without extra infra
Qualified Y/NCloser judgment. Becomes auto with call recording + LLM scoring (see step 07).
Call offer made (Y/N)Setter judgment when setters work the IG inbox manually. Becomes auto with a DM tool (Manychat / Instantly) or with disciplined GHL stage updates.
Show / no-showCloser flag if no Zoom/Meet attendance integration. Auto with the integration in step 05.
Recommended order of integration. Meta Ads + GHL + Stripe first — that single trio unlocks ~70% of the dashboard with maybe a day of work each. Then IG Graph for the organic side. Meeting attendance integration is a half-day. Call recording + LLM scoring last — it's the highest-leverage but also the most build-heavy.
