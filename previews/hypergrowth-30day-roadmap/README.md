## The seven pillars (James's structure)

Every client is diagnosed and tracked across these:

1. **DM Setting** — scripts, response rate, first-message, KPIs, setter management
2. **Content** — niche, positioning, TOF/MOF/BOF mix, page optimisation
3. **Ads** — strategy, spend, creatives, reading data, scaling without diminishing returns
4. **Team** — VA, setter(s), coach(es), hiring system
5. **Data & Ops** — what's tracked, what's read, what's actioned
6. **Founder** — working IN vs ON, time freedom, mindset
7. **Sales** — close rate, fear/objection/belief handling

These are the dashboard columns in Bite. Every weekly review walks the same seven.

## The universal 30-day spine

Same for every client. Order is non-negotiable — James enforces it in every handover.

### Week 1 — Diagnose, don't prescribe
- KPI tracking sheet installed Day 1 (Sam walks them through live, no "go set it up")
- Sam audits actual DMs and current setter situation
- Setter keep / retrain / replace decision by Day 5
- Setter (existing or new) plugged into Alessandro's Slack + training
- WhatsApp group spun up Day 0
- **Hard no:** ads, module dumps, pricing changes, coach hire

### Week 2 — Fix the front end
- DM scripts rebuilt in client's voice
- Daily outbound target (150–200/day per setter)
- Ad account **audited, not launched**
- Offer simplified (kill tiers — 29/57 of James's handovers tell the client to do this)
- Setter doing daily EOD report to Sam

### Week 3 — Turn on volume
- Ads launched **only if** DM KPIs above floor: ≥40% reply rate, ≥15% call-offer, ≥50% show, ≥40% close
- 4 creatives, £15/day each, follower/traffic ads
- Close-rate work begins (call reviews, objection handling)
- AOV restructure (move to 6-month PIF)

### Week 4 — Stabilise + plan next bottleneck
- Weekly review cadence locked in
- Coach hiring pipeline opened if client load justifies
- Retargeting layer added if cold ads working
- Founder time audit

## How the band changes what gets served

The spine is identical. The **weight, depth, and content** behind each step changes:

| Pillar | £5-15k client | £15-50k client |
|---|---|---|
| **DM Setting** | Build the system from scratch. No setter or weak setter. Scripts cringe. Foundational. | Audit B-player setters. Sharpen scripts. Enforce management. Loyalty-over-results conversation. |
| **Content** | Niche + positioning + 2nd page for results. Bio/highlights fix. Foundational. | Maintain. Solve peaks/troughs. Engagement consistency. |
| **Ads** | DEFERRED to week 3+. Belief work first. Predictability mindset. £0-1k → £1-5k range. | Already running £5-10k. Fix inefficiency. Push spend confidence. |
| **Team** | First setter hire. First coach decision (premium price + delegate vs do-it-themselves). | Upgrade from B to A players. Hiring system, not friend referrals. |
| **Data & Ops** | Get them tracking ANYTHING in every dept. | Tracking exists. Fix reading + action, not capture. |
| **Founder** | "IN vs ON" is the headline. Time freedom. | Less urgent. Mostly past this. |
| **Sales** | 30% close rate. Fear/objection/belief basics. | 50% close rate. Sharper fear/objection. |

For Bite content: same lesson library, different module-unlock order + different KPI floors per band.

## What Bite does

1. **Day 0 stage diagnostic** — pre-call intake form → assigns £5-15k or £15-50k band. Sam arrives at the human call already knowing the lane.
2. **Spine-locked content release** — Week 1 unlocks only Week 1 modules. Stops the #1 refund driver (overwhelm = "too many things at once" appears in 53/57 handovers).
3. **Seven-pillar dashboard** — KPIs auto-pulled from tracker. Sam and the client see the same view.
4. **Scheduled cross-team handoffs** — Bite books the Alessandro 1:1 and the Azri 1:1 at defined triggers, not "Sam will reach out."
5. **Drift alerts to Sam** — no booked call by day 10, no DMs sent in 3 days, WhatsApp silence >24h. Sam acts on signal, not on memory.

## The biggest insight from the data

**Sam's onboarding call is the same script every time, regardless of who's in front of him.** Across 9 calls covering everything from £4k pre-revenue to £30k multi-brand, he runs the same group-calls + setter-explainer + tracker-walkthrough opener. He doesn't diagnose. The framework he wrote — Stages 1/2/3 — is accurate and useful, but he doesn't use it as a diagnostic in real-time.

**The fix is structural, not training.** Make Bite run the diagnostic before the human call happens. Sam walks in knowing the client is £5-15k or £15-50k, knowing the broken pillar, knowing what Bite has already served. The 70-minute call compresses to 25. It becomes calibration, not setup.
