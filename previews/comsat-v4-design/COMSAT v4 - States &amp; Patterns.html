Anchor + flip rule Default anchor: top. If the tooltip's bounding box clips above the viewport, flip to bottom. If it clips on either side, shift the tooltip horizontally to stay 8 px from the viewport edge while keeping the arrow centered on the anchor element. The flip should be instant on first show (no animation), then sticky for the lifetime of that hover — don't oscillate.
Timing 500 ms before show on hover (filter out drive-by hovers). 100 ms before hide on hover-out, cancelable if the user re-enters the host within that window. No fade in/out — instant show, instant hide. Aria: aria-describedby + offscreen text mirror so screen readers don't lose the metadata.
Primary use · drawer hover Drawer tile hover → title row = full workspace title (e.g. "HPFHQ Consolidation Plan"). Meta row = `shorthand · workspace name · elapsed`. Lives on the right side of the tile (anchored right) so it doesn't fight the screen edge.
Secondary use · workflow dot hover Workflow slot hover → title = badge name (e.g. "Human in loop"). No meta row for badges — one line. Anchored top.

Sync·Cross-surface update cue

When a project's JSON updates from another surface (Nexus → COMSAT, VS Code → standalone), the receiving tile flashes a workspace-color wash over its surface. Uses ::after overlay so the underlying content is untouched.

SPEC
target: full tile/card surface (the workspace-color overlay sits at inset: 0)
color: color-mix(--ws 30%, transparent) — workspace tint, dialed back so it reads as "pulse" not "flash"
duration: 340 ms · cubic-bezier(.2,.7,.3,1)
easing: ease-out — full opacity at t=0, exponential decay to 0 at t=340 ms

audio: emits event:sync-received on each ping (after throttle).
12

Loading·Nexus ProjectView skeleton

Layout-preserving. Workspace color renders immediately (top border + drop shadow) so the user knows which project they're loading. Body content placeholders shimmer at 1.4 s linear infinite. Single shared shimmer pattern across every box.

Order of appearance (1) Project head renders immediately (workspace color top border + drop-shadow + title-as-skeleton). (2) All body cards render with their final dimensions + shimmer placeholders. (3) Cards swap to real content as data arrives, one at a time. The page never reflows during load.
Shimmer mechanics Base: text @ 8% fill. Overlay: linear-gradient(90deg, transparent, text @ 6%, transparent) sliding left-to-right at 1.4s linear infinite. Single keyframe, single pattern, reused everywhere. Three light-dark cycles ≈ 4.2s — if data takes longer than that, swap to a quieter pulse (no slide) to signal "still loading, but not stuck."
Stage pipeline preserves active marker Even during load, the active stage's copper tint renders so the user lands on the "which stage am I on" hint immediately — only the labels are skeleton.
Mid-write skeleton When the file is being saved (not loaded), the page replaces just the changing block (e.g. the Done-When list) with shimmer placeholders matching the previous content height. Header + other cards stay live. 800ms minimum show time so writes don't feel snappy-but-suspicious.

14

Drawer·Drag-to-reposition

Q6 — pure interaction spec, no visual cells (the drag is the interaction; there's nothing to render statically beyond the dragging state already shown in §07).

Grab zones Whole drawer is draggable when grabbed near (a) the top or bottom 12 px of the bar, OR (b) anywhere on the workspace-bar gutter (leftmost 6 px of any row). Tile rows themselves are not drag handles — clicks on tiles route to expand. Cursor changes to grab on those zones, grabbing while held.
Hot zone · snap targets 18 px from each screen edge counts as snappable. Dragging within a hot zone shows a 2 px accent-green ghost rail along that edge. All four edges show ghost rails on drag-start so all snap targets are visible up-front.
In-flight feedback Drawer detaches from its current edge, gains box-shadow: 0 12px 32px rgba(0,0,0,0.55), and shows a 1 px dashed outline indicating "in transit." Tile contents stay legible (drawer doesn't go ghost / semi-transparent).
Snap animation 220ms cubic-bezier(.2,.8,.2,1) — same easing as every other transition in the design system. The workspace bar reorients to the new edge (left edge → right edge means the bar swings to what's now the screen-edge side). Drawer geometry mirrors if needed.
Multi-monitor Crossing a display boundary while dragging re-renders ghost rails on the new display's edges. Snap takes effect on the display the cursor is on at release. Per-display dock edge memory — each monitor remembers its own preferred edge in the user's settings.
Cancel Esc during drag returns drawer to its previous position with no animation (instant revert — signals "cancel" clearly vs the smooth snap-to-target).
