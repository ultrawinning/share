01

Nexus·Mark·Concept

Hexagonal hub with central node. The hex is a network cell, the central dot is the convergence point — a nexus. Single-stroke geometry, scales clean from 16 px to 48 px, reads as a silhouette at any size.

geometryregular hexagon, vertex-up, viewBox 24×24
outer hexradius 9.5 (vertices at ±8.227, ±4.75)
inner hexradius 7.0 (vertices at ±6.062, ±3.5) · 50% opacity
central nodefilled circle r=2
stroke1.5 px at 24 px (scales with viewBox)
colorcurrentColor — one ink, no fills except the node
filemonochrome SVG, no embedded fonts, no clipping
02

Variants

Four variants of the mark family. Solid is the default. Outline drops the filled center node to outline-only (use on small icon bars). Monogram is just the mark, no chrome. Wordmark pairs the mark with the NEXUS lockup in Orbitron.

Use cases Solid — default app icon, ProjectView header, COMSAT chip launch. Outline — small monochrome contexts, VS Code activity bar variant (see §05). Monogram — favicon, sticker, sub-pixel renders where the inner hex would alias. Wordmark — splash screens, about dialogs, README headers.
When to drop the inner hex Below 20 px, the inner hexagon at 50% opacity stops reading and starts looking like noise. Use the Monogram variant (single hex + node) below 20 px. Solid variant kicks in at 20 px+.

Color rules On any dark or workspace-color background → white. On any light background → accent green (or workspace color, when the mark represents the workspace identity itself). On the COMSAT brand surface specifically → accent green. Don't render the mark in any other color — always one of (white, accent, workspace).

06

Done-When·Method icons

Four icons replacing the text method tags. Single style family: 12 px primary, 1.4 stroke at 12 px (scales with viewBox 16×16), square caps, no fills, no rounded corners. Each tag picks up its method's brand color from the existing token set.

12 · 20 · 36 px
MANUAL
Person silhouette — "a human verifies this." Head + shoulders, no body detail.
12 · 20 · 36 px
URL
Chain link — classic hypertext glyph. Two ovals joined at a 45° angle.
12 · 20 · 36 px
SHELL
Prompt chevron + cursor line — >_. Universally read as "terminal."
12 · 20 · 36 px
FILE
Document with corner fold — the most legible "file exists" glyph at small sizes.
Style rules viewBox 16×16, stroke 1.4, square line caps, miter joins. No fills, no rounded corners, no decorative elements. Each icon sits inside a 12×12 visual bbox with 2 px padding on all sides.
Color binding Each icon uses currentColor — the method tag chip sets its own color via class (.method-tag.url, etc) and the icon inherits. Manual = text, URL = signal cyan, Shell = accent green, File = warn amber.
Pixel-grid alignment at 12 px Geometry chosen so every stroke lands on a half-pixel boundary at 12 px target size (viewBox 16 = 0.75 scale). Avoids the half-pixel blur that kills small icons.
Drop-in replacement Existing CSS class .method on Done-When tags can keep its layout — only swap the text for the <svg> inline. Add a class to color-code. See §07 for the resulting list.
