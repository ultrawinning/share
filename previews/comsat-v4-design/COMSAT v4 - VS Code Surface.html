COMSAT V4 · VS CODE SURFACE R.01 STATUS-BAR BADGES · Q7
01

Concept

When a workflow badge is dragged from the COMSAT palette onto a VS Code window, it renders as a StatusBarItem in that window's status bar. VS Code constrains the API: foreground color is free, background only via two theme tokens, optional codicon prefix, tooltip is MarkdownString. Design works inside those constraints.

02

Anatomy

Single badge enlarged 3× for inspection. The base unit at real size is 22 px tall — the height of the VS Code status bar.

Why 3-letter codes Short enough to fit 4 badges in a single status-bar row without crowding the right-side built-ins (git sync, errors, position). Distinguishable at a glance: HMN / HLD / STBY / ATTN / CL / CX. Avoids single-character ambiguity ("H" could be human, hold, or hot).
Why color-on-color, not chip-on-chip VS Code lets us set foreground via StatusBarItem.color but only allows error/warning backgrounds. We use the foreground freely (workflow color text on the user's status-bar bg) and leave background to the user's theme. Looks native, never fights the OS chrome.

Workflow vs engine vs status Workflow badges (HMN/HLD/STBY) reflect user intent — pinned by drag. Engine pins (CL/CX) reflect routing preference — pinned by the user too, but the system honors them automatically. Status badges (ATTN) are system-set — user can't drag them, they appear when a condition is met and clear when resolved.
Why no workspace color in the badge text Each badge uses its workflow color, not the workspace's color. Workspace identity lives in the ws-tag (see §04) — that's the segment that wears the workspace tint. Badges = state, not identity. Mixing them would create double-encoded info.
04

Tooltip·Format

Each badge's tooltip uses a 3-line MarkdownString: name, state line, action hint. Standardised across all six badges so the user learns the rhythm once.

item.tooltip = new MarkdownString(
  `**${name}**\n` +
  `${stateLine}\n` +
  ${actionHint && `\n[${actionHint}](command:${cmd})`}
);
item.tooltip.isTrusted = true; // allow command links
namebold title — "Human in loop", "On hold", etc.
state lineone-line context — "Paused 18m ago", "Build failed: 2 type errors"
action hintcommand link — "Click to resume" / "Click to open Problems panel"

Workspace tag at lead position The ws-tag sits leftmost in our cluster (priority 200). It carries the NEXUS mark in workspace color + shorthand. Acts as the anchor between the workspace and its badge cluster. Click = open the project in Nexus.
ATTN gets a subtle background ATTN is the only badge with backgroundColor: 'statusBarItem.warningBackground' applied — using the one VS-Code-approved background to make system-set urgency unmissable. All others use plain foreground color on the user's theme bg.
Order rule ws-tag · engine pins · workflow badges · status badges. Codified via VS Code's priority (200 / 150–140 / 130–110 / 100). Stable across reloads; the engine you've pinned doesn't shuffle position when a workflow badge appears.
Truncation under width pressure If the editor's status bar runs out of room (lots of git, errors, warnings), VS Code right-truncates from the leftmost item. Our cluster is right-aligned and high-priority so it survives. If absolute width pressure hits, drop the labels first and keep just icons (codicons render at 14×14, ~20 px wide each).

Drag & drop entrypoint Badge palette opens from the COMSAT tile shield. Dragging a badge over a VS Code window highlights the window with an accent-green outline; releasing pins the badge for that workspace. To remove: right-click the badge → "Unpin from workspace."
Visibility persistence Pinned badges persist in the workspace's .json file (the same shared store COMSAT reads). On VS Code reload, badges reappear in the same order via stored priorities. Unpinning a badge removes it from the json entry.
Click handlers Each badge maps to a command. Defaults: HMN → open Nexus · HLD → resume workspace · STBY → no-op (informational only) · ATTN → open Problems panel · CL / CX → switch engine command. Command IDs land under comsat.badge.* namespace.
Theme compatibility Tested foreground colors hold contrast on the default Dark+, Light+, and Monokai themes. Dracula + Solarized Light may need a contrast-adjusted variant of the pink — document but don't ship until users hit it.
