Quick Start

Open Metabase at analytics.hpfhq.com. If that domain is not live yet, use the temporary Railway URL provided by Emil.

What you can use: dashboards, saved questions, filters, summaries, charts, exports, and your own saved views.
What you will see: curated reporting views only. These are prepared tables designed for reporting.
What you will not see: raw lead tables, email addresses, phone numbers, Slack payloads, notes, transcripts, or call links.

  2. Use the table Djermain - Email Sales Calls by Period.

  4. Use the columns Booked Calls, Showed Calls, Converted Calls, Cash Collected, and Contract Value.

Email attribution definition

The report counts rows as email-attributed when the source data points to email, newsletter, ConvertKit/Kit, or known email campaign patterns. The exact source logic is documented in Reporting - Data Catalog.

Important: This is a reporting view, not the raw leads list. If a row looks surprising, check Reporting - Data Catalog, Reporting - Data Freshness, and Verification - Djermain Email Funnel before making a decision from it.

Source And Validity

Every reporting view is connected to an internal source. Metabase should make it easy to inspect whether a metric is complete, current, and appropriate for the decision you are making.

Use these two questions first

Question What it tells you
Reporting - Data Catalog Source tables, row grain, included fields, excluded private fields, reliability notes, limitations, and validation method.
Reporting - Data Freshness Row count, earliest date, latest date, and freshness timestamp for each reporting view.
Verification - Dan Monthly Summary One row per visible monthly metric with value, source tables, source fields, calculation, reliability, caveat, and validation method.
Verification - Djermain Email Funnel One row per visible email funnel metric with value, source tables, source fields, calculation, reliability, caveat, and validation method.
Verification - Metric Gaps Metrics that are useful operationally but still need independent reconciliation before being called verified.

Current accuracy position

March and April dashboard values are live operational reporting from curated HPFHQ Supabase views. That means they are sourced, but most values are not independently verified yet.

Decision rule: A sourced number can be used for operating the business. A verified number has been reconciled to an independent system of record or cross-checked against a designated source of truth. For now, cash and contract values should be treated as operational USD fields until finance/sales reconciliation is complete.

Data safety rule

Metabase is connected through a read-only database role. It is intended for analysis, dashboards, and exports from curated views only. It should not be used to browse or export raw private lead data.

Read-only access Reporting schema only No raw PII tables Source notes included

Metric Verification

Use verification views when someone asks, "Is this accurate?" The answer should name the source, calculation, reliability, and caveat rather than just saying yes.

Reliability labels

Label Meaning
VERIFIED_STATIC_DEFINITION Static labels, period boundaries, or notes that are defined in the reporting migration.
SOURCED_NOT_INDEPENDENTLY_VERIFIED Comes from a named Supabase reporting source, but has not been reconciled to an independent system of record.
DERIVED_FROM_ATTRIBUTION_RULES Calculated from source or UTM rules. Useful operationally, but inherits attribution limitations.
NEEDS_FINANCE_RECONCILIATION Operational money field that should be checked against Stripe/Fanbasis before being treated as finance truth.
NEEDS_SALES_RECONCILIATION Operational contract field that should be checked against closer tracker or contract records before being treated as sales truth.

How to answer an accuracy question

  1. Open the relevant verification question.
  2. Filter to the month, period, or metric being discussed.
  3. Read the Source Tables, Source Fields, and Calculation columns.
  4. Read Verification Status and Caveats.
  5. If the metric says it needs reconciliation, treat it as operational reporting, not final truth.

Build Your Own Views

Metabase lets you create your own views without writing SQL. Start simple, then make the result prettier once the number is right.

Create a new question

  1. Click + New.
  2. Choose Question.
  3. Select the HPFHQ database.
  4. Choose a curated reporting view.
  5. Use Filter for date range, source, campaign, closer, setter, or outcome.
  6. Use Summarize to group by month, source, campaign, ad, closer, or setter.
  7. Click Visualization to switch from a table to a number, bar chart, or line chart.
  8. Click Save and place it in the right collection.

Make a dashboard prettier

  1. Open the dashboard and click Edit.
  2. Add saved questions as cards.
  3. Resize the most important numbers larger.
  4. Put trend charts below headline metrics.
  5. Keep source/validity cards at the bottom so the dashboard stays readable.
  6. Click Save.

When to ask Emil

  • You need a new raw data source added to reporting.
  • You need a new curated view that does not exist yet.
  • You see raw lead/private data inside Metabase.
  • A metric conflicts with HPFHQ cockpit or another official report.
