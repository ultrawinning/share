HPF Lead Flow — Full System Map

Every system hop from first touch to close. HPFHQ is source of truth. GHL is a mirror.

1
Typeform Submission
Lead fills out application. Webhook fires. HPFHQ creates/updates lead, assigns hpf_id, computes qualification tier. Parallel: Slack alert, Meta CAPI conversion, GHL contact enrichment (async).
Typeform→ HPFHQ leads→ Slack Meta CAPI GHL mirror
2
GHL Booking Webhook
Lead books call → GHL fires webhook → HPFHQ marks booked, maps ghl_contact_id. If no lead exists (ManyChat/direct), creates from GHL contact. Slack alert to #sales-calls-booked.
GHL Booking→ leads.booked=true→ Slack
2b
ManyChat (IG Organic)
Organic IG DM → ManyChat qual flow → webhook → HPFHQ creates lead (source=instagram, is_paid=false). Slack alert suppressed (fires before qual data captured).
ManyChat→ HPFHQ leads
3
Setter Triage
Setter reviews leads in HPFHQ portal. Fixes attribution, links setter_leads. GHL is read-only context — setter never edits GHL directly.
Setter Portal→ leads + setter_leads
4
Closer Workstation — Today
Reads GHL appointments + Fathom recordings + HPFHQ lead data. Identity resolution: contactId → hpf_id → lead. Cards show name, qual, phone, setter, Fathom link.
GHL Calendar+ Fathom+ leads→ Closer Cards
5
Closer EOD Submit
Outcome per call + reflection → submitted_at set → Slack digest to #sales-closer-eod. Follow-up leads enter pipeline table. GHL reflection fires stage changes (flag-gated).
EOD entries→ Slack EOD→ FU pipeline→ GHL stages
6
Closer Pipeline — Kanban
Hot | Nurture columns. Drag-and-drop, inline FU date editing, activity badges, objection filter. Moves update sales_followup_pipeline. GHL needed here for SMS/WhatsApp follow-up — closer opens GHL to message the lead.
Kanban↔ GHL SMS/WhatsApp
7
Payment → Close
Fanbasis or Stripe webhook → match by email → hpf_id → lead marked closed. Slack win to #sales-calls-closed. Meta CAPI Purchase. GHL → WON.
Fanbasis/Stripe→ leads.closed→ Slack CAPI GHL WON

Where GHL is Actually Needed vs. Mirror

FunctionCurrentlyGHL Required?
Booking calendarGHL calendarYes — no replacement without custom booking UI
SMS / WhatsApp / EmailClosers send via GHLYes — only outbound comms channel to prospects
Workflow automationsBooking confirmations, remindersYes — HPFHQ doesn't message prospects
Activity historyGHL tracks all touchpointsYes for now — only source of text/email timestamps
Pipeline viewMirrored from HPFHQNo — HPFHQ kanban replaces this
Contact databaseGHL contactsNo — HPFHQ leads table is source of truth
ReportingHPFHQ dashboardsNo — GHL just mirrors
Bottom line: GHL is irreplaceable for outbound communication (SMS, WhatsApp, email to prospects) and calendar booking. Everything else is already in HPFHQ. The GHL write-through (Phase 3.3.b, lib/ghl/reflection.ts) keeps GHL in sync as a read-only mirror. Replacing GHL's comms would require Sendinblue/Twilio for email+SMS + a custom booking UI — that's a platform-level lift, not a closer workstation feature. For now, the kanban links out to GHL when the closer needs to message someone.
