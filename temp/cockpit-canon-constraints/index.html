§1Executive summary #

The v2 plan is strategically correct — consolidate not rebuild, cockpit as new canon, phased sequencing, Overseer monitoring-only — but tactically thin on rule alignment. Three failure modes if we build as-currently-specified:

  1. Parallel metric registry conflict. CLAUDE.md and docs/extension-contracts.md both designate metric_definitions + lib/metrics.generated.ts as the canonical registry. v2 builds a parallel lib/cockpit/metrics/registry.ts with a parallel MetricDef interface. This is the Frankenstein trap.
  2. Helpers re-implement what lib/hq/queries.ts already does. getHqSnapshot() is a working KPI/pacing/projection/comparison engine for the /hq cockpit. v2 helpers would re-implement the same shape. Extend, don't duplicate.
  3. Lessons + integration rules silently ignored. Lead-date column is typeform_submitted_at not created_at (lessons-locked 2026-04-21). Hover tooltips for info-dense content are unreliable (lessons-locked 2026-04-07). Integration docs are required reading per CLAUDE.md non-negotiable rule. v2 plan addresses none of these.
The fix is plan amendment, not abandonment. The strategic direction stands. Tactics need to comply with what already exists.

3.2Pagination — gap confirmed #

No generic Supabase paginator exists. The 1000-row cap fix DOES need a new helper, but tightly scoped. Two options:

  • Option A: New lib/cockpit/paginate.ts with a single paginateAll<T>() function that loops .range() until empty. Genuinely new, justified.
  • Option B (recommended): Route every cockpit count through count: 'exact', head: true and never need pagination for tile values; only paginate the drill-down lead list (less frequent need). Less new infrastructure, smaller surface area.

3.4Design tokens — what exists #

lib/design-tokens.ts exports HPF_THEME, SPACING, RADIUS, SHADOW, MOTION, DEPT_COLORS, PRIORITY_COLORS, STATUS_COLORS, SOURCE_COLORS, PLATFORM_COLORS. Tailwind config maps these to classes like hpf-card, hpf-green, status-success.

Missing from Tailwind: greenGlow, greenFaint, borderDim, cardDeep — only available as raw hex via HPF_THEME in style={}. Tooltip + dictionary tab must use these (rule: no hardcoded hex, severity medium).
Conflict in CLAUDE.md vs codebase: Design system docs say green #2ED47A is "logo-only, not a button background, not a link color." Codebase uses green freely as active card borders, pacing bars, delta-up states, AuditCard metric color. v2 should follow codebase practice (green = accent), not the doc.
