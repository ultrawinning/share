Current pipeline

as of 2026-07-14
01Client paysLive

FanBasis (primary) / Stripe (legacy) → payments webhook

Claims charge → stamps closed_at, cash, contract_value on the lead → arms the onboarding watchdog (onboarding_status=pending, first-write-wins) → posts to the payment feed.

Orphan risk: matches lead by email. Payer email ≠ lead email → no close, no watchdog. Guard exists, but it's a real failure at this step.

02Onboarding watchdogLive

data-integrity cron

pending → escalates to alerted after ~72h. The safety net: "paid but still not in Trainerize" fires an alert.

03Contract + Trainerize setupGap

GHL + a human · not in HPFHQ

Contract sent and Trainerize account created externally / manually. HPFHQ only watches — it never sends the contract or provisions the account. contract_signed_at is never written.

04Trainerize signupLive

Zapier: "New Client Created" → trainerize/webhook

Creates the coaching_clients row (coach_id NULL) → clears the watchdog to onboarded → pings #coaching-new-clients to flag Sheena.

05Consult-form emailHeld

same signup event, gated on CONSULT_FORM_SEND_ENABLED

Code complete, idempotent on consultation_sent_at — but dark: the 3 env vars don't exist in prod. Has never fired once.

06Consult-form fillLive

public form → /api/f/submit → form_responses

Signed-token form; answers persist + biometric auto-fill. Works — but starved (no email → no inbound).

Hole: no coach notification on fill. A coach only sees it by opening the drawer.

07Welcome / Program SummaryManual

coach clicks generate → Claude drafts → copy-paste

Built, but there is no send in code at all — the coach copies the markdown and sends by hand.

08Coach assignmentLive

coaching-sync cron · daily 08:00 UTC

New client is born coach_id=NULL; the daily sync name-matches to team_roster. Shows "Unassigned" on the coach's own board up to ~24h.

09Check-ins + retentionLive

3-group check-in (manual) · retention_notes (human)

60-day retention = a human-filled tracker (Sheena). Data-quality crons flag stale/empty notes.

10NPS surveyGap

nps-trigger route · day-90 milestone

Double-dead: not scheduled in vercel.json (never runs), and blocked by the messaging switch even if hit. Targets day 90, not the 60-day tracker.
