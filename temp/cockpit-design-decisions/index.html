D8Pacing display + comparison delta locked research #

Locked: No pacing on percentages. Inline +/- delta vs comparison on every tile. UX research returned — four sub-decisions all confirmed against canonical patterns.

Comparison option set (D8.1)

Lock to four options: Previous Month (default), Same Month Last Year, Trailing 3-Month Average, vs Goal. Drop generic "previous period" — redundant at monthly cadence and confusing on custom ranges. Source: convergence across Looker Studio + Mixpanel + Amplitude + ChartMogul + Power BI.

Delta surface (D8.2)

Inline on the tile, both absolute and relative if space allows. Source: Datadog Query Value widget pattern. Anti-pattern (rejected): tooltip-only delta.

Percent vs percentage points (D8.3) — the "don't do dumb shit" rule

Rule: when the metric is itself a percentage (close rate, show rate, cancel rate), delta is in percentage points (pp), NOT percent. Close rate 28% → 32% = +4pp, not +14%. Most tools botch this by default — Mixpanel + Looker Studio both wrong out of the box.

Schema-level implementation: every metric row gets a metric_type column with values currency | count | rate | ratio. Rendering reads this:

  • metric_type=rate → delta computed as current - comparison, displayed as +4pp
  • metric_type=currency or count → delta computed as relative percent, displayed as +12%
  • metric_type=ratio → display as x multiplier delta

Direction-of-good (D8.4) — schema addition

Cancel rate trending DOWN is good. Refund rate down is good. CAC down is good. Green-up/red-down hardcoding is wrong for ~20–30% of ops metrics.

Schema-level implementation: every metric row gets a good_direction column with values up | down (default up). Rendering inverts color + arrow when down. Source: Mixpanel Metric Trees polarity field + Power BI Zebra BI Cards polarity.

Metrics to flag down in our Sheet: sales.cancelled, sales.no_shows, sales.quick_dqs, sales.cancel_rate, marketing.cost_per_lead, marketing.cost_per_call.

Comparison selector UI (D8.5)

Cockpit-level toggle (one control switches all tiles together) — Brandon glances and goes, single mental frame. Per-tile override capability in the data model from day one (you'll want it eventually), but the UI surfaces only the cockpit-level toggle in v1. Source: Looker Studio cockpit-level pattern; Datadog hybrid pattern.

Same Period Last Year — calendar irregularities (D8.6)

  • MTD vs SPLY: May 1–20, 2025 (calendar-locked, not rolling).
  • Calendar quarter: Q2 2026 vs Q2 2025 aligned by date.
  • Leap year Feb 29: clamp to Feb 28 (PostgreSQL DATE_ADD INTERVAL -1 YEAR does this by default — correct).
  • Do NOT implement WTD vs SPLY — week boundaries shift 1–2 days year-over-year, no clean comparison exists.
  • UI obligation: surface the exact comparison window ("Compare with: May 1–20, 2025") — don't make Brandon do the math.
Sheet schema additions needed: 2 new columns on every metric row — metric_type (currency|count|rate|ratio) and good_direction (up|down|N/A). I'll re-push the Sheet with these populated after your confirm. Quick to apply — already know the per-metric defaults.

D10Refresh cadence — with cost constraint locked #

Locked: Auto-refresh OK in principle but no API/cost burn. Refresh-on-tab-focus + server-side cache with short TTL.

Implementation:

  • Server-side cache on the cockpit page-level data fetcher — 5min TTL. Heavy queries (count-via-head, EOD aggregates, ad-snapshot rollups) hit cache. One actual DB query per metric per 5min regardless of how many users open the cockpit.
  • Client refresh on tab focus only (not timer polling). When you switch back to the cockpit tab from another tab, it re-fetches — which mostly hits cache, cheap. When the tab is idle in the background, zero requests.
  • Manual "refresh now" button bypasses cache (forces fresh fetch). For the case Brandon wants the latest before a call.
  • Per-source "as of" indicators at the top of the cockpit ("Ads: 6h ago · EOD: live · Payments: live"). Surfaces freshness per data class.
(My prior rec was timer-polling every 5 min — would have burned requests on idle tabs. Overruled.)
