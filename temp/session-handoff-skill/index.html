session-handoff

A Claude Code skill — wrap a build session into a cold-start-proof handoff so the next chat starts at full speed.

What it does

Type /handoff (or "wrap up", "carry this over") and Claude captures git state + open tasks + decisions into a structured doc and a copy-paste prompt for the next session. The core idea: every handoff must pass a cold-start test — a fresh window with zero memory, given only the handoff + the repo, must know why the work matters, where to find every fact, what it must not break, and what's a human decision vs. buildable now.

Install (≈30 seconds)

  1. Copy the skill text below (button).
  2. Create the file: 
    ~/.claude/skills/session-handoff/SKILL.md
  3. Paste, save. Restart Claude Code (or open a new session).
  4. Trigger it with /handoff at the end of a build session.

Skills live in ~/.claude/skills/<name>/SKILL.md. The YAML frontmatter at the top is what registers the trigger — keep it.
