CONSOLE

Your HPF Sales Operations Assistant — powered by Claude

What is this?

CONSOLE is an AI assistant that knows HPF sales operations. It has access to curated knowledge about GHL workflows, sales metrics, the payment pipeline, Typeform qualification, and your scorecard. It answers from those files — not from the internet, not from guessing.

When it doesn't know something, it says so and flags the gap for Emil. Over time, those gaps get filled and the system gets smarter.

Getting started

1 Open Claude Code
Go to claude.ai/code and sign in with your Claude Team account.
2 Connect the repo
You should see HPF-Console-Sales in the repo selector at the bottom. If you don't, connect your GitHub account first — you'll only see this one repo.
3 Start a session
Open a new session and type /start. The assistant will introduce itself and ask you 5 quick questions about your role. Answer honestly — your answers help Emil improve the knowledge base.
4 Ask away
Ask any sales ops question. If the answer is in the knowledge files, you'll get it with a source citation. If not, it'll tell you and flag it.

GHL Operations

Sales Metrics

Workflow Inventory

Stage map, metrics, targets, advancement criteria, weekly report format
HPF Context
Dan Go, client avatar, product lines, team structure, close definition

How to use it well

Do
  • Ask specific questions
  • Ask about GHL workflows and config
  • Ask for help drafting your weekly report
  • Ask about metric formulas and targets
  • Ask "how does X work?" about any sales process
Don't
  • Ask for API keys or webhook secrets
  • Ask it to write code or scripts
  • Ask about non-HPF topics
  • Paste passwords or sensitive credentials
  • Ask it to guess when it says "I don't know"

When it can't answer

If the assistant says "This isn't covered in my knowledge files" — that's working correctly. It will automatically save a knowledge request that Emil reviews weekly. The more you use it, the more gaps get identified and filled.

Escalation

If the assistant tells you to escalate to Emil, that means the question involves code, database, API integrations, webhook configuration, targets, compensation, or strategic decisions. Those are outside the ops knowledge base by design.
